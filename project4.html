My Works

담당 파트

1.지점 관리자

  • 지점 관리자 페이지에서 담당 지점의 상영 영화 관리(추가/삭제)
  • 담당 지점에서의 상영 일정 신청(최종 승인은 슈퍼관리자가 담당)

2.영화 예매

  • ajax로 페이지 일부분만 갱신
  • 예매 메인 페이지:영화/지점/날짜 선택 시 해당하는 상영 일정 출력, 일정 선택 가능
  • 좌석 선택 페이지:checkbox로 영화관 좌석 배치도와 유사하게 구현

3.결제

  • 아임포트 API를 사용하여 테스터 결제로 진행
  • 아임포트 서버로 결제를 요청(IMP.request_pay) 후 성공 응답 시 DB에 결제정보를 저장
  • 회원이 환불을 원할 시, DB에 저장된 결제 정보와 AccessToken을 발급받아 서버로 환불 요청

3.그 외

  • 극장: 극장 내 상영 일정 출력 시 상영관 명 기준 오름차순 출력
  • 회원-예매내역: dto 데이터 중 상영관,좌석정보 등 필요데이터 추가 출력 및 가독성 고려한 ui 수정

트러블 슈팅

1.아임포트 환불 처리 시 AccessToken은 필수

  • 이슈 : 환불 처리시 AccessToken값의 부재로 오류 발생.
  • 원인 : 결제와 다르게 AccessToken을 발급받고, 인수로 사용해야 한다.
  • 해결 : AccessToken을 생성 후 해당 값과 DB에 저장해 둔 필수값들을 사용하여 처리하였다.

2.극장 내 상영 일정 출력 시 데이터 순서가 보장되지 않는다.

  • 이슈 : 상영관 명(1관,2관,특별관..) 오름차순으로 출력하려 했으나 의도대로 되지 않음.
  • 원인 : 쿼리문의 결과값을 담는 과정에서 hashmap 을 사용해서 순서가 보장 되지 않음.
  • 해결 : linkedHashmap을 사용하여 오름차순의 결과를 담고, 출력하는데 성공하였다.

3.여러개의 json값을 js->controller로 전송 시 한개의 dto만 넘겨진다.

  • 이슈 : json배열:dto를 한개씩 매칭하여 총 3개의 dto를 사용하려 했으나 한개의 dto값만 받게됨.
  • 원인 : contoroller의 requestbody는 한 개의 dto만 받을 수 있다.
  • 해결 : 복합 dto 1개 안에 3개의 dto를 포함시켜 원하는 값을 모두 controller로 넘겨주었다.
